# 设置

VMark 的设置面板让你可以自定义编辑器的方方面面。使用 `Mod + ,` 或通过菜单栏中的 **VMark > 设置** 打开。

设置窗口的侧边栏按主题分组列出各章节 —— 最常用的章节在前，"关于"和"高级"在底部。更改立即生效 —— 没有保存按钮。

## 外观

控制视觉主题和窗口行为。

### 主题

从五种颜色主题中选择。活动主题以其色块周围的圆环标示。

| 主题 | 背景色 | 风格 |
|------|--------|------|
| White | `#FFFFFF` | 简洁、高对比度 |
| Paper | `#EEEDED` | 温暖中性色（默认） |
| Mint | `#CCE6D0` | 柔和绿色，护眼 |
| Sepia | `#F9F0DB` | 温暖米黄色，书本感 |
| Night | `#23262B` | 深色模式 |

### 语言

| 设置 | 描述 | 默认值 | 选项 |
|------|------|--------|------|
| 语言 | 更改菜单、标签和消息的界面语言，立即生效 | English | English、简体中文、繁體中文、日本語、한국어、Español、Français、Deutsch、Italiano、Português (Brasil) |

### 窗口

| 设置 | 描述 | 默认值 |
|------|------|--------|

| 自动隐藏状态栏 | 不与状态栏交互时自动隐藏 | 关闭 |

### Lint

| 设置 | 描述 | 默认值 | 选项 |
|------|------|--------|------|
| 启用 Markdown 检查 | 检查常见 Markdown 问题（损坏的链接、缺少 alt 文本、标题层级跳跃、未关闭的代码围栏等） | 开启 | 开启/关闭 |

完整规则列表和严重级别请参阅 [Markdown 检查](/zh-CN/guide/lint)。

### HTML 渲染

| 设置 | 描述 | 默认值 | 选项 |
|------|------|--------|------|
| 所见即所得中的原始 HTML | 控制是否在所见即所得模式下渲染原始 HTML 块 | 隐藏 | 隐藏、净化、净化 + 样式 |

::: tip
**隐藏** 是最安全的选项 —— 原始 HTML 块折叠且不渲染。**净化** 在去除危险标签后渲染 HTML。**净化 + 样式** 额外保留内联 `style` 属性。
:::

## 集成

MCP 服务器和 AI 提供商配置。

### MCP 服务器

MCP（模型上下文协议）服务器允许外部 AI 助手（如 Claude Code 和 Cursor）以编程方式控制 VMark。

| 设置 | 描述 | 默认值 |
|------|------|--------|
| 启用 MCP 服务器 | 启动或停止 MCP 服务器。运行时，状态徽章显示端口和已连接的客户端 | 开启（切换） |
| 启动时自动启动 | VMark 打开时自动启动 MCP 服务器 | 开启 |
| 自动批准编辑 | 不显示预览确认，直接应用 AI 发起的文档更改，请谨慎使用 | 关闭 |

服务器运行时，面板还会显示：
- **端口** —— 自动分配；AI 客户端通过配置文件发现端口
- **版本** —— MCP 服务器附属程序版本
- **工具/资源** —— 可用的 MCP 工具和资源数量
- **已连接的客户端** —— 当前连接的 AI 客户端数量

在 MCP 服务器部分下方，你可以一键将 VMark 的 MCP 配置安装到支持的 AI 客户端（Claude Desktop、Claude Code、Codex CLI、Gemini CLI）。

完整详情请参阅 [MCP 设置](/zh-CN/guide/mcp-setup) 和 [MCP 工具参考](/zh-CN/guide/mcp-tools)。

### AI 提供商

配置驱动 [AI 精灵](/zh-CN/guide/ai-genies)的 AI 提供商。同一时间只能激活一个提供商。

**CLI 提供商** —— 使用本地安装的 AI CLI 工具（Claude、Codex、Gemini）。点击 **检测** 扫描 `$PATH` 中可用的 CLI。CLI 提供商使用你的订阅计划，无需 API 密钥。

**REST API 提供商** —— 直接连接云端 API（Anthropic、OpenAI、Google AI、Ollama API）。每个都需要端点、API 密钥和模型名称。

各提供商的详细设置说明请参阅 [AI 提供商](/zh-CN/guide/ai-providers)。

## 格式

非默认格式适配器的可选开关，以及只读代码标签页的外部编辑器命令。

Markdown、纯文本和 YAML/YML **始终**已注册 —— 这是默认的稳定选项。其他所有适配器**默认关闭**，避免升级时出乎意料。切换开关后，注册表即时重建；已打开的标签页会以正确的适配器重新挂载，无需重启。

完整的格式列表及其预览说明，请参阅[支持的格式](/zh-CN/guide/formats)。

### 格式支持

| 开关 | 默认 | 启用内容 |
|---|---|---|
| **数据格式** | 关闭 | `.json`、`.jsonl`、`.toml` —— 分栏源码 + 可导航树。支持 `Cargo.toml`、`package.json`、`pyproject.toml` 的架构感知预览。 |
| **图表 & SVG** | 关闭 | `.mmd`（Mermaid）和 `.svg` —— 分栏源码 + 净化后的实时渲染。 |
| **HTML 预览** | 关闭 | `.html` 和 `.htm` —— 沙盒 iframe 预览（空 `sandbox=""` 允许列表、DOMPurify、CSP `<meta>`）。已通过 OWASP top-20 验证 —— 参见 [HTML 安全模型](/zh-CN/guide/formats#html-安全模型)。 |
| **代码查看器** | 关闭 | 12 种只读查看器（`.ts`、`.tsx`、`.js`、`.jsx`、`.py`、`.rs`、`.go`、`.css`、`.sh`、`.bash`、`.rb`、`.lua`）。在带语法高亮的查看器中打开，提供 **启用编辑** 和 **在外部编辑器中打开** 按钮。 |

当某类别关闭时，匹配的扩展名会回退到纯文本模式，文件仍然可以打开 —— 只是没有架构视图。

### 外部编辑器

对于只读代码标签页上的 **在外部编辑器中打开** 按钮，在此选择要启动的编辑器。可以是应用包（例如 `/Applications/Visual Studio Code.app`）或可执行文件。

图形界面设置优先于任何环境变量 —— 显式优于隐式。留空则使用环境变量回退链 `$VMARK_EXTERNAL_EDITOR → $VISUAL → $EDITOR → 平台默认`。完整的解析顺序和安全关口说明，请参阅[在外部编辑器中打开](/zh-CN/guide/formats#在外部编辑器中打开)。

### 一次性升级提示

首次升级至多格式支持后启动时，VMark 会显示一个无阻塞的提示，引导你前往 **设置 → 格式**。此提示每次安装只显示一次 —— 显示或关闭后不再出现。

## 快捷键

查看和自定义所有键盘快捷键。快捷键按类别分组（文件、编辑、视图、格式等）。

- **搜索** —— 按名称、类别或按键组合过滤快捷键
- **点击快捷键** 更改其绑定，按下新的组合键后确认
- **重置** —— 恢复单个快捷键的默认值，或一次性重置所有快捷键
- **导出/导入** —— 将自定义绑定保存为 JSON 文件，并在另一台机器上导入

完整的默认快捷键参考请参阅[键盘快捷键](/zh-CN/guide/shortcuts)。

## 终端

配置集成终端面板。使用 `` Ctrl + ` `` 打开终端。

| 设置 | 描述 | 默认值 | 选项 |
|------|------|--------|------|
| Shell | 使用哪个 Shell，需要重启终端才能生效 | 系统默认 | 系统上自动检测到的 Shell（如 zsh、bash、fish） |
| 面板位置 | 终端面板放置的位置 | 自动 | 自动（基于窗口纵横比）、底部、右侧 |
| 面板大小 | 终端占用可用空间的比例，拖动调整面板大小也会更新此值 | 40% | 10% 至 80% |
| 字号 | 终端中的文字大小 | 13px | 10px 至 24px |
| 行高 | 终端行间垂直间距 | 1.2（紧凑） | 1.0（紧密）至 2.0（超宽） |
| 光标样式 | 终端光标的形状 | 竖线 | 竖线、方块、下划线 |
| 光标闪烁 | 终端光标是否闪烁 | 开启 | 开启/关闭 |
| 选中即复制 | 自动将选中的终端文本复制到剪贴板 | 关闭 | 开启/关闭 |
| WebGL 渲染器 | 使用 GPU 加速渲染终端，如遇输入法输入问题请禁用，需要重启终端 | 开启 | 开启/关闭 |

有关会话、键盘快捷键和 Shell 环境的更多信息，请参阅[集成终端](/zh-CN/guide/terminal)。

## 关于

显示应用版本、网站和 GitHub 仓库链接，以及更新管理。

### 更新

| 设置 | 描述 | 默认值 |
|------|------|--------|
| 自动更新 | 启动时自动检查更新 | 开启 |
| 立即检查 | 手动触发更新检查 | — |

有可用更新时，会显示一张卡片，显示新版本号、发布日期和版本说明。你可以 **下载** 更新、**跳过** 此版本，或 —— 下载完成后 —— **重启以更新**。

## 高级

::: tip
高级部分默认隐藏。在设置窗口中按 `Ctrl + Option + Cmd + D` 可显示。
:::

开发者和系统级配置。

### 链接协议

| 设置 | 描述 | 默认值 |
|------|------|--------|
| 自定义链接协议 | 插入链接时 VMark 应识别的额外 URL 协议，以标签形式输入每个协议 | `obsidian`、`vscode`、`dict`、`x-dictionary` |

这让你可以创建如 `obsidian://open?vault=...` 或 `vscode://file/...` 这样的链接，VMark 会将其视为有效 URL。

### 性能

| 设置 | 描述 | 默认值 |
|------|------|--------|
| 同时保持两个编辑器活跃 | 同时挂载所见即所得和源码模式编辑器，以加快模式切换，会增加内存使用 | 关闭 |

### Workflow 引擎

| 设置 | 描述 | 默认值 | 选项 |
|------|------|--------|------|
| Workflow 引擎 | 为 `.github/workflows/` 下的 `.yml` / `.yaml` 文件启用 GitHub Actions workflow 查看器/编辑器。关闭时这些文件以普通 YAML 方式打开 | 关闭 | 开启/关闭 |
| 保留 YAML 格式 | 通过表单面板保存 workflow 编辑时，借助 CST 往返管线保留原 YAML 的注释、锚点、键序和空行。关闭时使用紧凑序列化器（更快但有损） | 开启 | 开启/关闭 |

完整功能介绍请参阅 [Workflow 查看器](/guide/workflow-viewer)。

### 平台特定

| 设置 | 描述 | 默认值 | 平台 |
|------|------|--------|------|
| 打开时清除 macOS 隔离属性 | 当打开带有 macOS 隔离属性（`com.apple.quarantine`）的文件时，在读取前剥除该属性。对从网络下载、本来会被 VMark 阻止打开的文件很有帮助 | 开启 | macOS |
| 将 Mac Option 视为 Meta（终端） | 在集成终端中将 macOS Option 键视为 Meta，emacs、tmux 等期待 Alt 前缀快捷键的工具需要此项 | 关闭 | macOS |

### 开发者工具

当 **开发者工具** 切换为开启时，会出现一个 **热退出开发工具** 面板，其中包含用于测试会话捕获、检查、恢复、清除和重启的按钮 —— 在开发过程中调试热退出行为时很有用。
